# Соответствие реализации тексту диплома (главы 1–2)

Документ для согласования главы 3 с уже написанными главами 1–2.

## Реализовано и можно описывать как «сделано»

1. **Архитектура модульного пайплайна** (раздел 2.1) — последовательность этапов в коде.
2. **Предобработка** (2.2) — очистка, нормализация, маскирование PII.
3. **Извлечение признаков** (2.2) — длина текста, URL/phone/email флаги, пунктуация.
4. **Гибридная детекция** (2.3) — правила + ML + граф + аномалии.
5. **Скоринг и пороги** (2.4) — взвешенная формула, три уровня риска.
6. **Безопасность** (2.6) — хэш автора, маски, audit log.
7. **Интерфейс расследования** — список инцидентов, карточка, подтверждение/отклонение.
8. **Экспериментальная база** — датасет, метрики ML, pytest, покрытие кода.

## Упрощено относительно раздела 1.5 (нормально для прототипа)

| В дипломе (1.5) | В прототипе |
|-----------------|-------------|
| API Twitter/VK, threat feeds | CSV + REST, поле `source` |
| OCR, гео, user-agent | Не реализовано |
| BERT / эмбеддинги | TF-IDF + LogReg |
| Очередь Celery/Kafka | Синхронная обработка в запросе |
| PDF/JSON экспорт отчётов | Только API JSON |
| SHAP/LIME | Показ 4 компонент скора и rule_flags |
| Переобучение по feedback | Статусы сохраняются, в ML не идут |
| Настройка правил без кода | Правила в `detection.py` |
| Push/email оповещения | Нет |
| RBAC, шифрование at-rest | Нет |

## Метрики для таблицы в дипломе

Запустить на сервере:

```bash
.venv/bin/python -m app.ml.train
.venv/bin/python -m pytest --cov=app -q
```

Указать: F1, precision, recall с обучения; число тестов; % покрытия `app/`.
